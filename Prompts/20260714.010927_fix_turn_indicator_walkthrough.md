# User Prompt
when game is waiting on BLACK to play, the message says 'White's turn'... is that right?

# Walkthrough: Fixing the Turn Indicator Bug

This document outlines the bug fix applied to the game's internal state machine that was causing the UI to display the incorrect player turn.

## Actions Taken
1. **Identified the Logic Flaw:** In `app/src/main/java/com/vypeensoft/reversi/game/GameManager.java`, whenever a player made a move, the `updateScores()` function (which tells the UI whose turn it is) was being executed *immediately* after placing the piece on the board, but *before* the internal state switched to the next player. 
2. **Reordered State Updates:** I removed the early call to `updateScores()` and instead injected it at the very bottom of the method chain.
3. **Result:** Now, when a piece is played, the game successfully changes hands (e.g. Black passes to White) *and then* alerts the UI, ensuring that if it is Black's turn to physically tap the screen, the UI correctly reads "Black's turn".
