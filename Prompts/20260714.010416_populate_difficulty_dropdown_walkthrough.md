# Walkthrough: Wiring Settings Activity

This document summarizes the fix to correctly populate and handle interactions with the Difficulty dropdown in the Settings screen.

## Actions Taken
1. **Adapter Creation:** The `spinnerDifficulty` widget was defined in the layout XML but lacked a data source. In `app/src/main/java/com/vypeensoft/reversi/activity/SettingsActivity.java`, I created an `ArrayAdapter` containing three difficulty levels: "EASY", "MEDIUM", and "HARD".
2. **Data Binding:** 
   - When the `SettingsActivity` loads, it now parses the `SettingsManager` and automatically sets the dropdown selection to match the user's currently saved preference (defaulting to MEDIUM).
   - An `OnItemSelectedListener` was attached to the spinner. Whenever the user taps a new difficulty level, it immediately writes that new string ("EASY", "MEDIUM", "HARD") back into the `SettingsManager.difficulty` property.
3. **Saving State:** Since the "Save and Exit" button already triggers the `SettingsManager.save()` function, the new difficulty selection will automatically be written to the `/sdcard/Vypeensoft/Reversi/settings/settings.json` file.

The dropdown is now fully functional and correctly alters the state of the game!
