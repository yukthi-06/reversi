# User Prompt
the game screen is full screen

# Walkthrough: Fixing Edge-to-Edge Layout Bleed

This document summarizes the changes made to fix the layout appearing as full screen despite the theme changes.

## Actions Taken
1. **Identified Edge-to-Edge Behavior:** Starting with Android 15 (API 35), the system enforces Edge-to-Edge display by default. This causes the application's root layout to draw underneath the translucent system status bar and navigation bar, creating the appearance of "full screen" mode.
2. **Applied `fitsSystemWindows`:** Added the `android:fitsSystemWindows="true"` property to the root `LinearLayout` in both `app/src/main/res/layout/activity_game.xml` and `app/src/main/res/layout/activity_main.xml`. 
3. **Result:** The system will now automatically apply padding to the root layout equal to the size of the system window insets (status bar and navigation bar). The game screen is now properly contained within the standard window boundaries and will no longer appear full screen.
