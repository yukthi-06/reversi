# Reversi Game Implementation Plan

## Goal Description
Build a fully playable, production-quality Reversi (Othello) Android game using Java, Android Views, and standard SDK up to API 35. The app will feature a custom View for the board, Minimax AI with Alpha-Beta pruning, JSON-based saving/loading of game state and settings to `/sdcard/Vypeensoft/Reversi/`, theme support, sound/animation support, and undo functionality.

## User Review Required
> [!IMPORTANT]
> The app needs to write to `/sdcard/Vypeensoft/Reversi/`. For Android API 35 (Android 15), writing to generic external storage locations using traditional `File` APIs requires `MANAGE_EXTERNAL_STORAGE` permission (All Files Access), as standard scoped storage only allows writing to app-specific directories without user interaction via SAF (Storage Access Framework). I will request `MANAGE_EXTERNAL_STORAGE` to strictly follow the requirement to write specifically to `/sdcard/Vypeensoft/Reversi/...`. Does this align with your expectations, or would you prefer using standard Scoped Storage/MediaStore/SAF APIs which may slightly alter the path or require user picker dialogues?

## Proposed Changes

### Setup & Gradle
- Update `gradle-wrapper.properties` to use Gradle 9.5.1.
- Update `build.gradle` (Project) and `app/build.gradle` for AGP compatible with 9.5.1, targeting API 35, Min SDK 24, Java 17.
- Add `Gson` dependency for JSON parsing.

### AndroidManifest.xml
- Add permissions: `MANAGE_EXTERNAL_STORAGE`, `READ_EXTERNAL_STORAGE`, `WRITE_EXTERNAL_STORAGE`, `VIBRATE`.
- Define activities: `MainActivity`, `GameActivity`, `SettingsActivity`, `AboutActivity`.

### Storage & Model Layer
- `JsonStorage`: Utility to handle reading/writing `savegame.json` and `settings.json` to `/sdcard/Vypeensoft/Reversi/`.
- `SettingsManager`: Singleton managing app settings (sound, difficulty, theme, etc.).
- `GameState`: Model representing the board, current player, score, move history.
- `Move`: Represents a move with coordinates and flipped pieces (for undo).

### Game Logic Layer (com.vypeensoft.reversi.game/board)
- `Board`: 8x8 array managing pieces, validating moves, checking for win conditions.
- `GameManager`: Orchestrates the game flow, turns, passing, and undo logic.
- `MoveValidator`: Helper for determining legal moves and pieces flipped.

### AI Layer (com.vypeensoft.reversi.ai)
- `AIPlayer` interface.
- `RandomAI`: Easy difficulty (random legal move).
- `MediumAI`: Prioritizes corners, avoids adjacent to corners.
- `MinimaxAI`: Hard difficulty using Minimax with Alpha-Beta pruning and heuristic evaluation (mobility, corners, piece diff). Runs on a background thread.

### UI & View Layer (com.vypeensoft.reversi.view/activity)
- `BoardView`: Custom View extending `View`. Handles touch events, draws grid, pieces, animations (canvas drawing, no bitmaps).
- `MainActivity`: Main Menu (New, Continue, Settings, About, Exit).
- `GameActivity`: Hosts the `BoardView`, score displays, undo/restart/menu buttons.
- `SettingsActivity`: UI to toggle settings (saved directly via `SettingsManager`).
- `AboutActivity`: Displays app/author info.
- `SoundManager`: Handles sound effects using `SoundPool`.
- `AnimationManager`: Utility for smooth piece flipping/placement animations.
