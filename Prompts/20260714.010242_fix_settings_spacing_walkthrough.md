# Walkthrough: Settings Layout Spacing

This document details the minor layout adjustment made to the Settings screen to prevent the content from clipping into the top edge of the device screen.

## Actions Taken
1. **Identified Edge-to-Edge Draw Issue:** Like the main menu and game screens previously, the new `activity_settings.xml` layout was defaulting to Android's Edge-to-Edge drawing, causing it to render underneath the transparent system status bar (where the clock/battery icons are).
2. **Applied System Window Fit:** I added `android:fitsSystemWindows="true"` to the root layout of the Settings page. This forces the Android OS to apply an automatic inset padding equal to the height of the status bar, pushing the content down to the safe area.
3. **Increased Padding:** To make it look even better and give it "a bit" more room, I increased the base padding of the root layout from `16dp` to `32dp`.

The fields are now safely lowered and no longer stuck to the very top edge of the screen!
