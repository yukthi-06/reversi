# Walkthrough: Disabling Full Screen Mode

This document summarizes the changes made to ensure the application does not run in full-screen mode and instead retains standard window decorations like the status and action bars.

## Actions Taken
1. **Identified Theme Settings:** Examined the primary theme in `app/src/main/res/values/themes.xml` (`Theme.Reversi`).
2. **Restored Action Bar:** The theme was originally configured to inherit from `Theme.MaterialComponents.DayNight.NoActionBar`. This has been updated to inherit from `Theme.MaterialComponents.DayNight.DarkActionBar`. This ensures the Android framework does not treat the activities as borderless/fullscreen and naturally retains the status bar and action title bar at the top of the screen.
3. **No Code Overrides Found:** Checked `AndroidManifest.xml` to verify no explicit fullscreen flags were set on the application or specific activities.

By making this change, the Reversi game will now display the standard Android OS bars, keeping it out of full screen mode.
